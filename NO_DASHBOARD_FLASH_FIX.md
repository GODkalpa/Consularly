## Key Code Changes

### 1. Added State Flag
```typescript
const [isAuthenticating, setIsAuthenticating] = useState(false)
```

### 2. Updated useEffect
```typescript
useEffect(() => {
  // Only redirect if NOT currently authenticating
  if (user && !profileLoading && !isLoading && !isAuthenticating) {
    redirectToDashboard()
  }
}, [user, profileLoading, isLoading, isAuthenticating, redirectToDashboard])
```

### 3. Enhanced Error Handling
```typescript
const handleEmailSignIn = async (e: React.FormEvent) => {
  setIsAuthenticating(true) // Block useEffect
  
  try {
    // ... authentication logic ...
    
    if (!response.ok) {
      await auth.signOut()
      setError(result.error)
      return // Stay on page
    }
    
    // Success - redirect
    router.push('/dashboard')
  } finally {
    setIsAuthenticating(false) // Re-enable useEffect
  }
}
```

## Timing Diagram

```

0ms:   User clicks "Sign In"
       isAuthenticating = true âœ…

50ms:  Firebase auth succeeds
       user state updates
       useEffect runs but BLOCKED by isAuthenticating

100ms: Session validation starts

150ms: Session validation fails (org mismatch)
       Returns 403 error

200ms: Error handler runs
       - Signs out user
       - Sets error message
       - Stays on login page

250ms: isAuthenticating = false
       useEffect runs but user = null (signed out)
       No redirect triggered

Result: User sees error on login page
        No dashboard flash âœ…
```

## Benefits

1. **No Visual Glitches**: User never sees dashboard
2. **Clear Feedback**: Error message displayed immediately
3. **Better UX**: No confusing redirects
4. **Secure**: Still validates organization access
5. **Fast**: No unnecessary navigation
