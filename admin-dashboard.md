"Create a robust system for a Master Admin to manage the entire SaaS platform for mock interview management, focusing on controlling access, managing organizations, and overseeing the platform's operations."

Primary Responsibilities of the Master Admin:

User Management:

Create and manage user accounts for students, organizations, and other admins.

Assign roles to users (e.g., student, organization, admin).

View a list of all users with filters to view organizations and students separately.

Edit or delete user accounts when necessary.

Organization Management:

Create new organizations (visa consultancies, agencies, etc.) and assign them a mock test quota (e.g., 100 tests per month).

View all organizations and their test usage.

Edit organization details (e.g., contact info, subscription plan).

Delete organizations or suspend access if needed.

Quota Management:

Set and update quotas for mock tests for each organization.

Track how many mock tests each organization has consumed in the current month.

Set rules for replenishing quotas (e.g., monthly reset, customizable).

Warn organizations if they are approaching their quota limit.

Platform Analytics:

View platform-wide statistics, such as:

Total number of students and organizations.

Total mock tests taken by students across the platform.

Test usage by organization.

Other high-level metrics to monitor platform health.

Billing & Subscription:

Track and manage billing for organizations (e.g., subscription tiers, payments).

Assign subscription plans to organizations based on their mock test quota and features.

Generate payment invoices for organizations.

Upgrade or downgrade organizations based on their subscription plan or usage.

Global Settings:

Configure global platform settings (e.g., currency, language, branding).

Set the platform's default mock interview questions and scenarios.

Enable or disable features like AI feedback, body language analysis, etc.

Support & Communication:

Handle user support tickets or queries from students and organizations.

Send notifications to students and organizations (e.g., quota alerts, test results, upcoming changes).

Manage email templates for communication (e.g., welcome emails, password resets).

Security & Privacy:

Manage security settings (e.g., two-factor authentication, access permissions).

Ensure data privacy compliance, such as GDPR or other relevant regulations.

Monitor activity logs to track platform access and user actions.

Testing & Reporting:

View detailed logs of all actions performed on the platform, including user logins, organization updates, test assignments, etc.

Generate reports for internal review on the usage of mock tests, student performance, and platform engagement.
